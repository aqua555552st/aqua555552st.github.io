4.Postman(接口测试工具)

基本介绍

  1. Postman是一款功能强大的用于发送HTTP请求的测试工具
  2. 做WEB页面开发和测试的人员常用工具
  3. 创建和发送任何HTTP请求(Get/Post/Put/Delete…)

快速入门

要求：使用Postman向http://www.baidu.com发出get请求，得到返回的html格式数据

STEP1：点击file，再点击new，选择HTTP Request。

STEP2：指定url和请求方法

STEP2：点击Send

使用Postman完成Controller测试

使用Postman，完成对前面编写的UserHandler方法的请求

案例1 — 常规

  1. 确定请求的地址url：http://localhost:8080/springmvc/user/buy

  2. 请求方式：Post

  3. 确定请求的参数：无

  4. 确定Header有没有特殊的指定

applicationContext-mvc.xml

<!--1.配置 注解自动扫描包-->
<context:component-scan base-package="com.lhy.web"/>

<!--2.配置 视图解析器-->
<bean class="org.springframework.web.servlet.view.InternalResourceViewResolver">
    <!--配置 属性前缀prefix和后缀suffix-->
    <property name="prefix" value="/WEB-INF/pages/"/><!--从web目录开始-->
    <property name="suffix" value=".jsp"/>
    <!--根据UserServlet的login()方法，return "login_ok"；-->
    <!--就是转发到 /WEB-INF/pages/login_ok.jsp-->
</bean>

登录界面login.jsp

<%@ page contentType="text/html;charset=UTF-8" language="java" %>
<html>
<head>
    <title>登录界面</title>
</head>
<body>
<h3>登录界面</h3>
<%--回顾javaweb web工程路径专题
    action="login":表示url是 http://localhost:8080/springmvc/login
    action="/login":表示url是 http://localhost:8080/login
--%>
<form action="user/buy" method="post">
    u:<input name="username" type="text"> <br/>
    p:<input name="password" type="password"><br/>
    <input type="submit" value="登录">
</form>
</body>
</html>

处理器/控制器UserHandler.java

@RequestMapping(value = "/user")
@Controller//Userhandler就是一个 处理器/控制器
public class UserHandler {
    /**
     * 解读
     * 1.method = RequestMethod.POST:表示请求buy目标方法必须是post
     * 2.RequestMethod 四个选项:POST GET PUT DELETE[后面详解]
     * 3.SpringMVC控制器 默认支持GET/POST两种方法
     *
     * 4.@RequestMapping同时标注在类和方法上
     * -> 请求的url就是：/类请求值/方法请求值，即：http://localhost:8080/工程路径/user/buy
     * @return
     */
    @RequestMapping(value = "/buy", method = RequestMethod.POST)
    public String buy() {
        System.out.println("购买商品");
        return "success";
    }
}

返回界面success.jsp

<%@ page contentType="text/html;charset=UTF-8" language="java" %>
<html>
<head>
    <title>操作成功</title>
</head>
<body>
    <h1>恭喜，操作成功~</h1>
</body>
</html>

运行成功

案例2 — 带请求参数

  1. 确定请求的地址url：http://localhost:8080/springmvc/user/find

  2. 请求方式：Get

  3. 确定请求的参数：bookId=100

  4. 确定Header有没有特殊的指定

applicationContext-mvc.xml

<!--1.配置 注解自动扫描包-->
<context:component-scan base-package="com.lhy.web"/>

<!--2.配置 视图解析器-->
<bean class="org.springframework.web.servlet.view.InternalResourceViewResolver">
    <!--配置 属性前缀prefix和后缀suffix-->
    <property name="prefix" value="/WEB-INF/pages/"/><!--从web目录开始-->
    <property name="suffix" value=".jsp"/>
    <!--根据UserServlet的login()方法，return "login_ok"；-->
    <!--就是转发到 /WEB-INF/pages/login_ok.jsp-->
</bean>

Postman控制页面

处理器/控制器UserHandler.java

@RequestMapping(value = "/user")
@Controller//Userhandler就是一个 处理器/控制器
public class UserHandler {
    /**
     * params = "bookId" 表示请求该目标方法时，必须给一个bookId参数，值没有限定
     * search(String bookId)：url携带的bookId值将作为search的参数
     * @param bookId
     * @return
     */
    @RequestMapping(value = "/find", method = RequestMethod.GET, params = "bookId")
    public String search(String bookId) {
        System.out.println("查询到的bookId为：" + bookId);
        return "success";
    }
}

运行成功
