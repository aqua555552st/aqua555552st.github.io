发表于2024-02-06|更新于2024-02-07|C语言
|字数总计:1.3k|阅读时长:6分钟|阅读量:

函数调用

本文使用如下源程序对函数的嵌套调用进行讲解：

stack.c

#include <stdio.h>

#define N 5

int abs(int num) {
    if(num < 0)
        return num * (-1);
    return num;
}

void get_arr_abs_sum(int * arr, int len) {
    int sum = 0;

    for(int i = 0; i < len; i++) {
        sum += abs(arr[i]);
    }

    printf("arr's abs sum = %d\n", sum);

    return;
}

int main(int argc, char * argv[]) {
    int arr[N] = {-1, 2, -3, 4, -5};

    get_arr_abs_sum(arr, N);

    return 0;
}

进程的虚拟地址空间

分析main函数

step1：先保存存栈基bp和栈帧sp，为main函数预分配48B栈空间

79:	55                   	push   %rbp
7a:	48 89 e5             	mov    %rsp,%rbp
7d:	48 83 ec 30          	sub    $0x30,%rsp #0x30即48B地址空间

step2：在x86-64中，函数的第一个参数通常被传递给 %edi 寄存器；函数的第二个参数通常被传递给 %rsi 寄存器

81:	89 7d dc             	mov    %edi,-0x24(%rbp)
84:	48 89 75 d0          	mov    %rsi,-0x30(%rbp)

step3：初始化arr数组

88:	c7 45 e0 ff ff ff ff 	movl   $0xffffffff,-0x20(%rbp) #arr的首地址
8f:	c7 45 e4 02 00 00 00 	movl   $0x2,-0x1c(%rbp)
96:	c7 45 e8 fd ff ff ff 	movl   $0xfffffffd,-0x18(%rbp)
9d:	c7 45 ec 04 00 00 00 	movl   $0x4,-0x14(%rbp)
a4:	c7 45 f0 fb ff ff ff 	movl   $0xfffffffb,-0x10(%rbp)

step4：即将进行函数调用，先将get_arr_abs_sum(int * arr, int len)的参数放入寄存器中。%esi存放的是len；%rax和%rdi存放的是arr的地址

ab:	48 8d 45 e0          	lea    -0x20(%rbp),%rax
af:	be 05 00 00 00       	mov    $0x5,%esi
b4:	48 89 c7             	mov    %rax,%rdi

main调用get_arr_abs_sum

step1：call指令实现函数跳转&返回：

  1. 将call指令的下一条地址入栈
  2. %rip指向main+0x43处
b7:	e8 00 00 00 00       	callq  bc <main+0x43>

step2：get_arr_abs_sum函数开始执行。push %rbp将%rbp指向栈顶，且内容为上层函数的栈基址bp。再开辟32B的栈空间作为get_arr_abs_sum函数的栈帧。

get_arr_abs_sum调用abs结束

假设此时是for循环的最后一次调用

step1：abs执行ret指令：

  1. 释放栈空间，再将栈顶的返回地址get_arr_abs_sum_call_next_ip从栈中弹出，
  2. 并写入%rip寄存器中
18:	c3                   	retq

main调用get_arr_abs_sum结束

step1：get_arr_abs_sum执行ret指令：

  1. 释放栈空间，再将栈顶的返回地址main_call_next_ip从栈中弹出，
  2. 并写入%rip寄存器中
