静态库与共享库

发表于2024-02-04|更新于2024-02-05|C语言
|字数总计:1.9k|阅读时长:7分钟|阅读量:

静态库与共享库

所谓"程序库"，就是包含了数据和执行码的文件。其不能单独执行，可以作为其他程序的一部分完成某些功能。

库的存在，可以使程序模块化，加快程序的再编译，可以实现代码重用，可以使得程序便于升级。

程序库可分为静态库(static library)和共享库(shared object)。两者本质区别：该库是否被编译进目标（程序）内部。

静态库

这类库在编译的时候会直接整合到目标程序中，所以利用静态函数库编译成的文件会比较大。应用于对空间要求低，时间要求高的程序中。

  • 优点：编译成功的可执行文件可以独立运行，而不再需要向外部要求读取函数库的内容
  • 缺点：相同的库文件数据可能在内存中被加载多份，消耗系统资源，浪费内存；函数库更新，需要重新编译

应用实例—生成静态库

/*add.c*/
int add(int a, int b) {
    return a + b;
}
/*sub.c*/
int sub(int a, int b) {
    return a - b;
}
/*div.c*/
int div(int a, int b) {
    return a / b;
}

step1：对add.c和sub.c和div.c三个文件进行编译，得到3个可重定位目标文件

step2：使用ar工具将3个可重定位目标文件生成为1个静态库libmymath.a。

step3：**在math.c程序编译过程中使用静态库libmymath.a**。

#include <stdio.h>

int main(int argc, char * argv[]) {
    int a = 4, b = 6;

    printf("%d + %d = %d\n", a, b, add(a, b));
    printf("%d - %d = %d\n", a, b, sub(a, b));
    printf("%d / %d = %d\n", a, b, div(a, b));
    return 0;
}

隐式声明

在C语言中，函数在调用前不一定非要声明。如果没有声明，那么编译器会自动按照一种隐式声明的规则，为调用函数的C代码产生汇编代码。

单纯的编译上述math.c源代码，并没有任何报错，只是在链接阶段若找不到名为add/sub/div的函数体则报错。之所以编译不会报错，是因为C语言规定，对于没有声明的函数，自动使用隐式声明。

编译math.c源代码，加上警告信息gcc math.c libmymath.a -o math -Wall：

[root@lhyEdu01 staticLib]# gcc math.c libmymath.a -o math -Wall
math.c: 在函数‘main’中:
math.c:6:2: 警告：隐式声明函数‘add’ [-Wimplicit-function-declaration]
  printf("%d + %d = %d\n", a, b, add(a, b));
  ^
math.c:7:2: 警告：隐式声明函数‘sub’ [-Wimplicit-function-declaration]
  printf("%d - %d = %d\n", a, b, sub(a, b));
  ^
math.c:8:2: 警告：隐式声明函数‘div’ [-Wimplicit-function-declaration]
  printf("%d / %d = %d\n", a, b, div(a, b));
  ^

编译通过！并没有任何报错，只是在链接阶段若找不到名为add/sub/div的函数体则报错。

之所以编译不会报错，是因为C语言规定，对于没有声明/定义的函数，自动使用隐式声明。相当于变成了如下代码：

#include <stdio.h>

//编译器的隐式声明:int function_name(int, ... parameter_type);
//默认返回值为int类型，对函数参数不关心，若函数实际返回值为其他类型，则链接时报错
//隐式声明的函数原型与库函数完全相同，则链接运行没有问题(如本例)
int add(int, int);
int sub(int, int);
int div(int, int);

int main(int argc, char * argv[]) {
    int a = 4, b = 6;

    printf("%d + %d = %d\n", a, b, add(a, b));
    printf("%d - %d = %d\n", a, b, sub(a, b));
    printf("%d / %d = %d\n", a, b, div(a, b));
    return 0;
}

显示声明

隐式声明给程序员带来了各种困惑，给程序的稳定性带来了非常坏的影响。一般使用显示声明，将函数定义/声明放入头文件中，在程序中引入头文件：

头文件mymath.h

/*
 * 加入的三行：头文件守卫
 *多个c程序都包含该头文件引用，编译时带来大量声明冲突
 * 加入头文件首位：避免多次包含
 * */
#ifndef _MYMATH_H_
#define _MYMATH_H_

int add(int, int);
int sub(int, int);
int div(int, int);

#endif

源程序math.c

#include <stdio.h>
#include "mymath.h"

int main(int argc, char * argv[]) {
    int a = 4, b = 6;

    printf("%d + %d = %d\n", a, b, add(a, b));
    printf("%d - %d = %d\n", a, b, sub(a, b));
    printf("%d / %d = %d\n", a, b, div(a, b));

    return 0;
}

动态库

动态函数库在编译的时候，在程序里只有一个“指向”的位置而已，也就是说当可执行文件需要使用到函数库的机制时，程序才会去读取函数库来使用。应用于对时间要求低，空间要求高的程序中。

  • 优点：可实现不同进程间的资源共享；动态库升级简单，只需要替换库文件，无需重新编译应用程序
  • 缺点：加载速度比静态库慢；发布程序需要提供依赖的动态库

应用实例

step1：对add.c和sub.c和div.c三个文件进行编译，得到3个可重定位目标文件(.o)

注意：动态库所需的可重定位目标文件与静态库所需的可重定位目标文件不同！

  • 静态库所用的.o文件中的调用函数地址：编译时生成(相对于main的相对寻址)
  • 动态库所用的.o文件中的调用函数地址：运行时生成！

因此编译时需要加上参数：-fPIC!!!!

[root@lhyEdu01 dymamicLib]# gcc -c add.c -o add.o -fPIC
[root@lhyEdu01 dymamicLib]# gcc -c sub.c -o sub.o -fPIC
[root@lhyEdu01 dymamicLib]# gcc -c div.c -o div.o -fPIC

step2：使用gcc -shared -o libxxx.so xxx.o xxx.o制作动态库

[root@lhyEdu01 dynamicLib]# gcc -shared -o libmymath.so add.o sub.o div.o

step3：编译可执行程序时，指定所用的动态库gcc myth.c -o a.out -lmymath -L ./ 头文件 。

-l：指定库名（libxxx.so的xxx是库名）

-L：指定库路径

[root@lhyEdu01 dynamicLib]# gcc math.c -o math.out -lmymath -L./ mymath.h

step4：math.out运行…失败！

下文分析运行失败原因

原因：链接器&动态链接器

链接器&动态链接器的关系：张三和张三丰、雷锋和雷峰塔，没有关系

  • 链接器：工作于链接阶段，工作时需要 -l 和 -L
  • 动态链接器：工作于程序运行阶段，工作时需要提供动态库所在目录位置。通过改变环境变量：export LD_LIBRARY_PATH=动态库所在路径；或者动态库使用绝对路径

方法1：通过改变环境变量

[root@lhyEdu01 dynamicLib]# export LD_LIBRARY_PATH=/unix/dynamicLib
#注意：环境变量只是暂时生效，终端重启后失效！
#永久改变需要更爱配置文件:vim ~/.bashrc

方法2：动态库使用绝对路径

扩展讲解：程序的地址空间

细节请参考：chapter7—链接 | aqua555552st’s script

代码演示

#include <stdio.h>
#include <stdlib.h>

int init_val = 10;//已初始化数据
int uninit_val;//未初始化数据

int main(int argc, char * argv[], char * env[]) {
    const char * s = "abcd";//只读常量

    printf("代码区(.text):%p\n", main);
    printf("只读常量区(.rodata):%p\n", s);
    printf("初始化数据(.data):%p\n", &init_val);
    printf("未初始化数据(.bss):%p\n", &uninit_val);

    int * p1 = (int *) malloc(sizeof(int) * 10);
    int * p2 = (int *) malloc(sizeof(int) * 10);
    int * p3 = (int *) malloc(sizeof(int) * 10);
    int * p4 = (int *) malloc(sizeof(int) * 10);
    //堆区
    printf("heap addr(p1):%p\n", p1);
    printf("heap addr(p2):%p\n", p2);
    printf("heap addr(p3):%p\n", p3);
    printf("heap addr(p4):%p\n", p4);

    //栈区
    printf("stack addr(p1):%p\n", &p1);
    printf("stack addr(p2):%p\n", &p2);
    printf("stack addr(p3):%p\n", &p3);
    printf("stack addr(p4):%p\n", &p4);

    //命令行参数地址
    printf("命令行参数地址:%p\n", argv[0]);

    //环境变量地址
    printf("环境变量地址:%p\n", env[0]);

    return 0;
}
