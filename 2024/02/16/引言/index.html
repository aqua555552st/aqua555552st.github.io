Spring核心学习内容

  1. IOC(Inversion Of Controller控制反转)：管理 Java对象
  2. AOP(切面编程)
  3. JdbcTemplate：是Spring提供的一套访问数据库的技术(应用性强)
  4. 声明式事务：基于IOC/AOP实现事务管理(理解较困难)

Spring几个重要概念

Spring可以整合其他框架

Spring的2个核心概念:IOC&AOP

传统开发模式：JdbcUtils/反射

  • 程序员编写程序，在程序中读取配置信息
  • 创建对象，new ObjectXxx() / 反射创建对象
  • 使用对象完成任务

IOC开发模式：[EmpAction EmpService EmpDao Emp]

  • Spring根据配置文件(xml/注解)，创建对象，并放到容器(ConcurrentHaspMap)中，并完成对象间的依赖
  • 当需要创建某个对象实例时，就直接从容器中获取即可
  • 程序员更关注相应业务(而非new 对象)

Spring核心价值

通过配置，给程序提供需要使用的web层[Servlet(Action/Controller)] / Service / Dao / [JavaBean/entity] 对象

快速入门

在项目下创建lib目录，在包中将以下5个.jar包复制并加入目录(add as library)

左键src-> New -> XML Configuration File -> Spring Config -> beans.xml

beans.xml文件

<!--配置Monster对象
        1.class:类的全路径
        2.id:该类创建的对象的 变量名-->
<bean class="com.lhyedu.spring.bean.Monster" id="monster01">
    <!--
        <property></property>:配置类的 属性标签
        1)name:属性名
        2)value:赋值-->
    <property name="monsterId" value="100"/>
    <property name="name" value="牛魔王"/>
    <property name="skill" value="牛魔王拳"/>
</bean>

Monster类

注意：反射创建对象时会用到==无参构造器==与==set方法==，定义类时必须写上！！！

package com.lhyedu.spring.bean;
/**
 * javaBean/Entity
 */
public class Monster {
    private String monsterId;
    private String name;
    private String skill;

    //无参构造器一定要写，反射时需要使用无参构造器！！！
    public Monster() {
    }

    public Monster(String monsterId, String name, String skill) {
        this.monsterId = monsterId;
        this.name = name;
        this.skill = skill;
    }

    //set方法一定要写，反射时用这个方法初始化类的属性！！！
    public void setMonsterId(String monsterId) {
        this.monsterId = monsterId;
    }

    public void setName(String name) {
        this.name = name;
    }

    public void setSkill(String skill) {
        this.skill = skill;
    }

    @Override
    public String toString() {
        return "Monster{" +
                "monsterId='" + monsterId + '\'' +
                ", name='" + name + '\'' +
                ", skill='" + skill + '\'' +
                '}';
    }
}

测试

package com.lhyedu.spring.test;

import com.lhyedu.spring.bean.Monster;
import org.junit.jupiter.api.Test;
import org.springframework.context.ApplicationContext;
import org.springframework.context.support.ClassPathXmlApplicationContext;


public class SpringBeanTest {
    @Test
    public void testFirst() {
        //1.创建IOC容器对象
        ApplicationContext ioc =
                new ClassPathXmlApplicationContext("beans.xml");

        //2.提供getBean获取对应对象，参数是<bean>的id
        Monster monster01 = ioc.getBean("monster01", Monster.class);
        System.out.println(monster01);
    }
}

类加载路径

问题：为什么

ApplicationContext ioc =
        new ClassPathXmlApplicationContext("beans.xml");

即==可读取beans.xml文件？==

classPath()：来获取==类加载路径，即：运行时的实际工作路径==

public class SpringBeanTest {
    @Test
    public void classPath() {
        //获取："类加载路径"
        File f = new File(this.getClass().getResource("/").getPath());
        System.out.println(f);
    }
}
