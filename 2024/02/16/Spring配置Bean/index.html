Spring配置Bean

[TOC]

以Monster类为例

package com.lhyedu.spring.bean;

public class Monster {
    private Integer monsterId;
    private String name;
    private String skill;

    //无参构造器一定要写，反射时需要使用无参构造器！！！
    public Monster() {
    }

    public Monster(Integer monsterId, String name, String skill) {
        this.monsterId = monsterId;
        this.name = name;
        this.skill = skill;
    }

    //set方法一定要写，反射时用这个方法初始化类的属性！！！
    public void setMonsterId(int monsterId) {
        this.monsterId = monsterId;
    }

    public void setName(String name) {
        this.name = name;
    }

    public void setSkill(String skill) {
        this.skill = skill;
    }

    @Override
    public String toString() {
        return "Monster{" +
                "monsterId='" + monsterId + '\'' +
                ", name='" + name + '\'' +
                ", skill='" + skill + '\'' +
                '}';
    }
}

基于xml文件配置

1.创建Bean对象

①通过类型获取bean

要求ioc容器中的同一个类的bean只能有一个，否则会抛出NoUniqueBeanDefinitionException！

应用场景：比如XxxAction/Servlet/Controller在一个线程中只需要一个对象实例(单例)

<bean class="com.lhyedu.spring.bean.Monster">
    <property name="monsterId" value="200"/>
    <property name="name" value="白骨精"/>
    <property name="skill" value="吃人"/>
</bean>
public class SpringBeanTest {
    @Test
    public void setBeanByConstructor() {
        ApplicationContext ioc =
                new ClassPathXmlApplicationContext("beans.xml");

        Monster bean = ioc.getBean(Monster.class);
        System.out.println(bean);
    }
}
Monster{monsterId='2', name='蜘蛛精', skill='吐丝'}

5.工厂模式获取bean对象

①静态工厂获取bean对象

在spring的ioc容器，可以通过静态工厂获取bean对象。

静态工厂类：MyStaticFactory

package com.lhyedu.spring.factory;
import com.lhyedu.spring.bean.Monster;
import java.util.HashMap;
import java.util.Map;

//静态工厂类，可以返回Monster对象
public class MyStaticFactory {
    private static Map<String, Monster> monsterMap;

    //使用 静态代码块 进行初始化
    static {
        monsterMap = new HashMap<>();
        monsterMap.put("monster_01", new Monster(100, "黄袍怪", "一阳指"));
        monsterMap.put("monster_02", new Monster(200, "九头金雕", "如来神掌"));
    }

    //[重点！]提供一个方法返回Monster对象
    public static Monster getMonster(String key) {
        return monsterMap.get(key);
    }
}

beans.xml

<!--配置monster对象，通过静态工厂获取-->
<!--重点！！！
    1.class标签：填写 静态工厂类的全类名
    2.factory-method标签：填写获取对象的方法-->
<bean id="m1" class="com.lhyedu.spring.factory.MyStaticFactory" factory-method="getMonster">
    <!--constructor-arg:factory-method方法的参数-->
    <constructor-arg value="monster_01"/>
</bean>
public class SpringBeanTest {
    @Test
    public void getBeanByStaticFactory() {
        ApplicationContext ioc =
                new ClassPathXmlApplicationContext("beans.xml");

        Monster m1 = ioc.getBean("m1", Monster.class);
        System.out.println(m1);
    }
}
Monster{monsterId='100', name='黄袍怪', skill='一阳指'}

③通过FactoryBean获取对象(重点)

在Spring的ioc容器，通过FactoryBean获取bean对象(重点)。

通过创建 ：类 implements FactoryBean。维护一个Map和key属性==，==并实现以下三个方法：

  1. getObject()：返回 Map属性中与keyVal对应的 对象
  2. getObjectType()：返回 Map属性中value的.class对象
  3. isSingleton()：是否以单例形式保存bean对象

MyFactoryBean类

package com.lhyedu.spring.factory;
import com.lhyedu.spring.bean.Monster;
import org.springframework.beans.factory.FactoryBean;
import java.util.Map;

public class MyFactoryBean implements FactoryBean<Monster> {
    private String key;
    private Map<String, Monster> monster_map;

    {
        monster_map = new HashMap<>();
        monster_map.put("monster03", new Monster(100, "黄袍怪", "一阳指"));
        monster_map.put("monster04", new Monster(200, "九头金雕", "如来神掌"));
    }
    
    public void setKey(String key) {
        this.key = key;
    }

    //返回 Map属性中与keyVal对应的 对象
    @Override
    public Monster getObject() throws Exception {
        return monster_map.get(key);
    }

    //返回 Map属性中value的.class对象
    @Override
    public Class<?> getObjectType() {
        return Monster.class;
    }

    //是否以单例形式保存bean对象
    @Override
    public boolean isSingleton() {
        return true;
    }
}

beans.xml

<!--配置Monster对象，通过FactoryBean获取-->
<!--class：指定使用的FactoryBean
    name="key"：标识属性key; value="monster04":key的值-->
<bean id="monster_04" class="com.lhyedu.spring.factory.MyFactoryBean">
    <property name="key" value="monster04"/>
</bean>
public class SpringBeanTest {
    @Test
    public void getBeanByFactoryBean() {
        ApplicationContext ioc =
                new ClassPathXmlApplicationContext("beans.xml");

        Monster my_monster_05 = ioc.getBean("my_monster_05", Monster.class);
        System.out.println(my_monster_05);
    }
}
Monster{monsterId='100', name='黄袍怪', skill='一阳指'}

1.bean配置信息重用(继承)

在spring的ioc容器中，提供了一种继承的方式来实现bean配置信息的重用。==>parent标签

beans.xml

    <!--配置Monster对象-->

3.bean的单例&多例

以以下测试文件为例：

package com.lhyedu.spring.bean;

public class Cat {
    private Integer id;
    private String name;

    public Cat() {
        System.out.println("构造器被调用..");
    }

    public void setId(Integer id) {
        this.id = id;
    }

    public void setName(String name) {
        this.name = name;
    }

    @Override
    public String toString() {
        return "Cat{" +
                "id=" + id +
                ", name='" + name + '\'' +
                '}';
    }
}
public class SpringBeanTest {
    @Test
    public void getBeanBySingleton() {
        ApplicationContext ioc =
                new ClassPathXmlApplicationContext("beans.xml");

        for (int i = 0; i < 3; i++) {
            System.out.println(ioc.getBean("cat0", Cat.class));
        }
    }
}

单例模式：在spring的ioc容器，在默认是按照单例创建(scope="singleton")的，即配置一个bean对象后，ioc容器只会创建一个bean实例。

beans.xml

<!--默认情况下，scope="singleton"(单例)-->
<bean id="cat0" class="com.lhyedu.spring.bean.Cat">
    <property name="id" value="1"/>
    <property name="name" value="小花猫"/>
</bean>

运行结果：

构造器被调用..//只创造了一个Cat对象
Cat{id=1, name='小花猫'}
Cat{id=1, name='小花猫'}
Cat{id=1, name='小花猫'}

多例模式：如果希望ioc容器配置某个bean对象，是以多个实例形式创建的则可以通过配置scope="prototype"

beans.xml

<!--scope="prototype"(原型)，每次getBean都创建一个新对象-->
<!--懒加载默认为false ==> lazy-init="false"-->
<bean id="cat0" class="com.lhyedu.spring.bean.Cat" scope="prototype" lazy-init="true">
    <property name="id" value="1"/>
    <property name="name" value="小花猫"/>
</bean>

补充：lazy-init(懒加载)：ioc容器初始化时不创建，getBean方法调用时才创建

运行结果：

构造器被调用..//创建多个Cat对象
Cat{id=1, name='小花猫'}//且可以看出：Cat对象创建是在getBean方法调用时，而非ioc初始化时

Cat{id=1, name='小花猫'}

4.bean的生命周期(重点)

bean对象的创建是由JVM完成的，然后执行如下方法：

  1. 执行构造器
  2. 执行setter方法
  3. 调用bean的初始化方法(需要配置init-method标签)
  4. 使用bean
  5. 当容器关闭时，调用bean的销毁方法(需要配置destroy-method标签)

House类

package com.lhyedu.spring.bean;

public class House {
    private String name;

    public House() {
        System.out.println("House() 构造器");
    }

    public void setName(String name) {
        System.out.println("setXx()方法被调用..");
        this.name = name;
    }

    public void init() {
        System.out.println("House init()..");
    }

    public void destroy() {
        System.out.println("House destroy()..");
    }
}

beans.xml

<!--init-method="init":指定init函数为bean初始化方法
    destroy-method="destroy":指定destroy函数为bean销毁方法-->
<bean id="house" class="com.lhyedu.spring.bean.House" 
      init-method="init" destroy-method="destroy">
    <property name="name" value="北京豪宅"/>
</bean>
public class SpringBeanTest {
    @Test

                new ClassPathXmlApplicationContext("beans.xml");

        House house = ioc.getBean("house", House.class);
        System.out.println("使用bean = " + house);

        //ioc运行类型：ClassPathXmlApplicationContext(有close方法)
        //ioc编译类型->向下转型，再调用close方法
        ((ClassPathXmlApplicationContext) ioc).close();
    }
}

后置处理器(涉及AOP先不细讲)

  • 在spring的ioc容器中，可以配置bean的后置处理器（本质：一个bean对象，作用比较特殊）
  • 该处理器(/对象)会在bean初始化方法(init-method)调用前和初始化方法调用后被调用
  • 程序员可以在后置处理器中编写自己的代码

House类

package com.lhyedu.spring.bean;

public class House {
    private String name;

    public House() {
        System.out.println("House() 构造器");
    }

    public void setName(String name) {
        System.out.println("setXx()方法被调用..");
        this.name = name;
    }

    public void init() {
        System.out.println("House init()..");
    }

    public void destroy() {
        System.out.println("House destroy()..");
    }
}

后置处理器(implements BeanPostProcessor并要重写两个方法)

package com.lhyedu.spring.bean;

import org.springframework.beans.BeansException;
import org.springframework.beans.factory.config.BeanPostProcessor;

//后置处理器
public class MyBeanPostProcessor implements BeanPostProcessor {
    /**
     * 什么时候被调用：在bean的init-method方法前会被调用
     * @param bean：传入的在ioc容器中 配置的bean对象
     * @param beanName：传入的在ioc容器中创建/配置bean的id
     * @return 程序员对传入的bean进行修改/处理，再返回
     * @throws BeansException
     */
    @Override
    public Object postProcessBeforeInitialization(Object bean, String beanName)
            throws BeansException {
        System.out.println("postProcessBeforeInitialization被调用。bean=" + beanName);
        return bean;
    }
    /**
     * 什么时候被调用：在bean的init-method方法后会被调用
     * 其他同上
     */
    @Override
    public Object postProcessAfterInitialization(Object bean, String beanName)
            throws BeansException {
        System.out.println("postProcessAfterInitialization被调用。bean=" + beanName);
        return bean;
    }
}

beans.xml

<!--重点：配置后续处理器类bean->该bean初始化后作用于其他bean的初始化(切面编程AOP)-->
    <bean class="com.lhyedu.spring.bean.MyBeanPostProcessor" id="mbpp"/>
    
    <!--包含后置处理器处理的bean的生命周期-->
    <bean id="house" class="com.lhyedu.spring.bean.House"
          init-method="init" destroy-method="destroy">
        <property name="name" value="北京豪宅"/>
    </bean>
public class SpringBeanTest {

    public void BeanPostProcessorTest() {
        ApplicationContext ioc =
                new ClassPathXmlApplicationContext("beans.xml");

        House house = ioc.getBean("house", House.class);

        ((ClassPathXmlApplicationContext) ioc).close();
    }
}

bean的自动装配(高频面试题)

在spring的ioc容器，可以实现自动装配bean

1.基于xml

  • autowire="byType"：在创建bean对象时，通过 类型 给属性自动完成引用

    >1)比如OrderService对象有 private OrderDao orderDao
>2)就会在容器中找 OrderDao类型 的对象。有->自动装配；多个同类型->报错

  • autowire="byName"：在创建bean对象时，通过 bean的id 给属性自动完成引用

    >1)比如OrderService对象有 private OrderDao orderDao
>2)再看属性的setXxx()方法的xxx(重点)
>3)如果没有->则装配失败

OrderDao类

package com.lhyedu.spring.dao;
//Dao层
public class OrderDao {
    //方法
    public void saveOrder() {
        System.out.println("保存 一个订单...");
    }
}

OrderService类

package com.lhyedu.spring.Service;

import com.lhyedu.spring.dao.OrderDao;
//Service层
public class OrderService {
    //引用到Dao类为属性
    private OrderDao orderDao;

    public OrderDao getOrderDao() {
        return orderDao;
    }

    public void setOrderDao(OrderDao orderDao) {
        this.orderDao = orderDao;
    }
}

OrderAction类

package com.lhyedu.spring.web;

import com.lhyedu.spring.Service.OrderService;
//Controller层
public class OrderAction {
    //引用到Service类为属性
    private OrderService orderService;

    public OrderService getOrderService() {
        return orderService;
    }

    public void setOrderService(OrderService orderService) {
        this.orderService = orderService;
    }
}

beans.xml（自动装配bean属性：autowire标签）

<!--配置OrderDao对象-->
<bean class="com.lhyedu.spring.dao.OrderDao" id="orderDao"/>
<!--配置OrderService对象，但不对属性进行<property>引用赋值->自动装配-->
<bean class="com.lhyedu.spring.Service.OrderService" id="orderService" autowire="byType"/>
<!--配置OrderAction对象，但不对属性进行<property>引用赋值->自动装配-->
<bean class="com.lhyedu.spring.web.OrderAction" id="orderAction" autowire="byName"/>
public class SpringBeanTest {
    @Test
    public void setBeanByAutowried() {
        ApplicationContext ioc =
                new ClassPathXmlApplicationContext("beans.xml");

        OrderAction oa = ioc.getBean("orderAction", OrderAction.class);
        OrderService os = oa.getOrderService();
        System.out.println(os);//验证OrderAction是否成功装配OrderService
        System.out.println(os.getOrderDao());//验证OrderService是否成功装配OrderDao
    }
}

2.基于注解

基于注解配置bean，也可实现自动装配，使用的注解是：@Autowired或@Resource

重点：除了泛型依赖注入的情况用：@Autowired；其他大多数情况下都用：@Resource

①@Autowired

UserService类

package com.lhyedu.spring.component;

import org.springframework.stereotype.Service;
//@Service：表示是一个处理业务逻辑的类，通常用于Service类
@Service
public class UserService {
    public void hi() {
        System.out.println("UserService hi()~");
    }
}

UserController类

package com.lhyedu.spring.component;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Controller;
//@Controller：标识是一个控制器，通常用于Servlet
@Controller
public class UserController {
    //自动装配
    //1)在IOC容器中查找带装配组件的类型，如果有唯一bean匹配(按类型)->装配
    //2)若带装配类型在ioc容器有多个，则根据带装配属性属性名作为id查找
    @Autowired
    private UserService userService;

    public void sayOk() {
        System.out.println("UserController 的sayOk()~");
        userService.hi();//调用Service层属性的方法
    }
}

beans.xml

<context:component-scan base-package="com.lhyedu.spring.component"/>
<bean class="com.lhyedu.spring.component.UserService" id="service"/>

    @Test
    public void setProByAutowired() {
        ApplicationContext ioc =
                new ClassPathXmlApplicationContext("beans.xml");

        UserController userController = ioc.getBean("userController", UserController.class);
        userController.sayOk();
    }
}

package com.lhyedu.spring.component;

import javax.annotation.Resource;

//@Controller：标识是一个控制器，通常用于Servlet
@Controller
public class UserController {

        userServiceWuYiYi.hi();//调用Service层属性的方法

}
